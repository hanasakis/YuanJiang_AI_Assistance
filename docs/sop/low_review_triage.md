# 差评订单分诊处理 SOP

**文档编号**: SOP-REVIEW-002
**版本**: v1.0
**适用范围**: 客服 + 品控 + 运营

---

## 1. 差评定义

差评 = 评价分数 ≤ 2 星的订单。

### 1.1 关键指标

| 指标 | 公式 | 数据来源 |
|------|------|----------|
| low_review_rate | 差评订单数 / 总评价订单数 | review_risk_metrics |
| avg_review_score | AVG(review_score) | review_risk_metrics |
| negative_review_rate_pct | 差评占比 × 100 | review_risk_metrics |

### 1.2 风险等级

**P0 紧急（24h 客服介入）**:
- low_review_rate > 50%
- 或单一商品收到 ≥ 5 条差评且含安全关键词（破损、漏液、冒烟）

**P1 重要（48h 品控复审）**:
- low_review_rate 30% ~ 50%
- 或 avg_review_score < 2.5

**P2 关注（周度品控抽查）**:
- low_review_rate 10% ~ 30%

**P3 观察**:
- low_review_rate < 10% 但近期上升

## 2. 差评分诊流程

### 2.1 自动分类

将差评内容按关键词自动分类：

| 类别 | 关键词 | 责任方 |
|------|--------|--------|
| 物流问题 | 慢、延迟、破损、包装差 | 承运商 / 卖家 |
| 商品质量 | 坏、故障、与描述不符、假货 | 卖家 / 品控 |
| 服务态度 | 态度差、不回复、拒绝退款 | 卖家客服 |
| 价格问题 | 贵、不值、降价 | 市场定价 |

### 2.2 分诊路由

```
物流问题 → 转卖家 + 承运商投诉工单
商品质量 → 转品控部门 + 冻结该商品
服务态度 → 转卖家培训 + 扣信用分
价格问题 → 转市场部 + 不做处罚
```

### 2.3 回复模板

**物流延迟**: "非常抱歉您的订单未能按时送达。我们已通知承运商加快配送，并为您申请了 X 元补偿。您的订单编号是 {order_id}，当前物流状态为 {status}。"

**商品质量**: "很抱歉您收到的商品有问题。我们已为您安排退换货，退款将在 3-5 个工作日内到账。同时我们已对该商品启动品控复查。"

---

## 3. 差评趋势告警

### 3.1 卖家级

连续 2 周 low_review_rate 上升 → 触发 P1 告警
单周内 3 条以上差评 → 触发人工复审

### 3.2 品类级

某品类整体 low_review_rate > 20% → 触发品类级品控巡检
某品类连续 4 周差评率上升 → 建议调整品类策略

---

## 4. 常见案例

### 案例 1：差评突增—商品批次问题

某电子产品本周收到 12 条差评（上周 1 条），全部指向同一批次。
品控抽检发现该批次存在焊接不良。
**处理**: 冻结该批次 → 召回已售商品 → 通报供应商。

### 案例 2：差评率虚高—样本量过小

某新卖家 small_seller 仅有 5 条评价，其中 3 条因物流差评。
差评率 60%，但 5 条的样本量不具统计意义。
**处理**: P3 观察，不触发告警。标记为"样本不足"。
